issueの修正を実装し、レビューを経てプルリクエストを作成する。

対象issue: $ARGUMENTS

## 手順

1. **issue内容の確認**
   - `gh issue view` でissueの詳細を取得する
   - 修正対象のファイルと対応方針を把握する

2. **ブランチ作成**
   - mainブランチから `fix/issue-{番号}-{簡潔な説明}` の形式でブランチを作成する

3. **実装**
   - issueの内容に基づいて修正を実装する
   - CLAUDE.mdのコーディング規則に従う

4. **コミット**
   - 変更内容をコミットする

5. **Codexレビュー**
   - `/codex` スキルを使ってコードレビューを実施する
   - 指摘事項があれば修正し、再コミットする

6. **プルリクエスト作成**
   - リモートにプッシュし、PRを作成する
   - PR本文にissue番号を含め、`Closes #番号` で自動クローズを設定する
   - PR本文に「Codex CLIによるコードレビュー済み」であることを明記する
