# 1on1-tool 要件定義（Requirements）

## 1. プロジェクト概要

### 1.1 目的
1on1における対話の質を最大化させる。日常の「余白」に生まれる本音や感情を安全に可視化し、上司と部下が表面的な業務報告を超えた深い信頼関係を築くための補助ツールとする。

### 1.2 基本思想（プロダクトの魂）
- **本音は日々の「余白」で：** 業務時間中の報告ではなく、ふとした瞬間の感情を拾う。
- **非可逆性の担保：** 「後で整える」を禁止。その瞬間の鮮度と生々しさを優先する。
- **点ではなく線：** 個別の発言の是非ではなく、全体的な「傾向・流れ・温度感」に価値を置く。
- **評価ではなく関係性の補助：** 人事評価のための道具ではなく、対話を円滑にするための「温度計」。

## 2. ユーザー定義・関係性

### 2.1 登場人物
- **入力者（部下）：** 日々の感情ログを入力し、AI要約に注釈を加える主体。自身のアウトプット（要約＋注釈）は永続的に閲覧可能。
- **閲覧者（上司）：** 公開されたAI要約と注釈を閲覧し、1on1の対話に活用する主体。
- **システム管理者：** システム開発会社（運用担当者）を想定。保守・運用のために全データを閲覧・編集でき、システム日付の操作権限を持つ特権ユーザー。

### 2.2 関係性の定義
- ペアリングは管理者により設定され、変更可能とする。
- AがBの、BがCの、CがAの上司となるような「循環構造」を許容する。
- 上下関係が単純な階層構造でない組織形態にも対応する。

## 3. 機能要件

### 3.1 日々の入力フェーズ（Daily Log）
- **入力項目（業務報告ではない）：**
  - やったこと（文脈）
  - プラスの感情
  - マイナスの感情
  - 本音ベース（独り言・生のログ）

- **入力ルール：**
  - **日付変更線：** 翌日の正午（12:00）までを当日分として許容。
  - **非可逆性：** 確定後の修正・削除は不可。
  - **秘匿性：** 確定後の再閲覧（本人含む）、および他者による原文ログの閲覧は一切不可。

- **入力形式：** テキスト入力および音声入力に対応。
  - **音声の扱い：** テキスト変換完了後、音声ファイル自体は即座に破棄（物理削除）する。

### 3.2 アウトプット生成・注釈フェーズ（Analysis & Annotation）
- **生成条件：** 5日分以上のログ蓄積 ＋ 本人による実行ボタン押下。
- **AI要約内容：** 日々の入力に基づいた「感情の傾向」「継続して出ているテーマ」「温度感や変化」。

- **注釈（アノテーション）機能：**
  - AIの要約文自体は書き換え不可とし、本人が「補足・注釈」を追記する形式。
  - 文字数制限を設け（例：1項目200文字以内）、簡潔な補足を促す。
  - AIの意見と本人の注釈を併記して表示する。

- **公開フロー：** 部下が内容を確認し「OK」を出した後に上司が閲覧可能となる。

### 3.3 閲覧フェーズ（Viewing）
- **閲覧制限：**
  - **上司への割り当て：** 上司は割り当てられたアウトプットのみ閲覧可能。
  - **閲覧期限：** 公開から **1週間（168時間）限定**。期限を過ぎると上司は閲覧不可となる。

- **表示形式：** AI要約に加え、部下が加えた「注釈」をハイライト表示し、対話の重点トピックとして可視化する。

### 3.4 システム管理者権限（例外対応）
- **全データアクセス：** 入力者本人の生ログ（原文）を含む全データの閲覧が可能。
- **データ操作：** 各データの削除、編集が可能。
- **日付操作：** **システム上の日付（マスター日付）の編集・操作が可能。**（過去日の入力救済や運用テスト用）

## 4. AIキャラクター設計：通訳者（ブリッジ・アドボケイト）
AIは「部下の専属エージェント」として振る舞う。

- **基本スタンス：** 部下に対しては全肯定。上司に対しては「部下のパフォーマンス向上のための建設的アドバイザー」。
- **翻訳の方向性：** 「主語」を個人ではなく「仕組みや環境」に置き換え、角を立てずに本質を伝える。
- **変換方針：** 生々しい不満を、上司が「改善のヒント」として受け取れる前向きな言葉に変換する。

## 5. 非機能要件
- **セキュリティ：** 生ログデータはDB保存時にアプリケーション層で暗号化（AES-256等）を施す。
- **可用性：** AI解析等の高負荷処理はキュー（非同期処理）により実行する。
- **監査ログ：** 管理者による特権操作（生ログ閲覧・日付操作等）はすべて履歴を記録する。

## 6. フェーズ2以降の検討事項（Backlog）
- メンタルリスクや強いネガティブ感情に対する自動アラート機能。
- 継続利用のためのフィードバック（入力日数や抽象化された感情グラフの表示）。
- 定時の入力リマインド通知機能。
